# スキルの一元管理・運用ガイド

本ドキュメントでは、この `skills` リポジトリを複数の開発端末で共有して運用する方法と、別の端末に分散しているAIスキル（エージェントスキル）を収集して一元化する手順を解説します。

---

## 1. 他の端末でこのスキルを運用する方法

この共通リポジトリ（`skills`）を別のPCでも使用できるようにするための導入・運用手順です。

### 1.1 リポジトリの同期（Gitの利用）

1. 新しい端末で本リポジトリを `git clone` します。
   ```bash
   git clone <本リポジトリのURL>
   ```

2. **（推奨）自動更新の設定**
   本リポジトリには、Windows 起動時に自動で最新のスキルを取得する仕組みが用意されています。
   - `automation/sync_skills.bat` を実行するか、スタートアップに登録することで、常に最新の状態でスキルを利用できます。
   - 手動で即座に更新したい場合は、`automation/auto_update.py` を実行してください。
   ```bash
   python automation/auto_update.py
   ```

### 1.2 Python パッケージ（汎用関数）としての運用

`text` や `automation` 内にあるPythonスクリプトを共有ライブラリとしてシステム全体で利用する場合は、インストールが必要です。

1. クローンした `skills` ディレクトリに移動します。
2. 開発（編集）モードでpipインストールします。
   ```bash
   pip install -e .
   ```

> [!TIP]
> `-e` オプションをつけてインストールすることで、リポジトリ内のコードを変更するとすぐにシステム全体に変更が反映されます。

### 1.4 スキルを有効化（アクティベート）する方法

収集したエージェントスキルをAIアシスタントに実際に使わせるためには、そのアシスタントが監視している特定のディレクトリに対象のスキルフォルダが存在している必要があります。

#### A. 作業中のプロジェクトに導入する場合（推奨）

特定のプロジェクトでのみそのスキルを使いたい場合は、プロジェクトルートにリンクを作成します。

1. 作業中プロジェクトのルートディレクトリに移動します。
2. `.codex/skills/` ディレクトリがない場合は作成します。
3. 今回の `skills` リポジトリにある `agent/` 内の目的のスキルを、`.codex/skills/` へコピーまたはシンボリックリンクを貼ります。
   ```powershell
   # PowerShellでのシンボリックリンク例（管理者権限が必要な場合があります）
   New-Item -ItemType SymbolicLink -Path ".codex\skills\skill-folder-name" -Value "C:\data\GitHub\skills\agent\skill-folder-name"
   ```

#### B. ツール全体でグローバルに使えるようにする場合

AIエディタ自体が参照している「共通スキルフォルダ」がある場合（例: `~/.codex/skills/`）、そこへリンクまたはコピーを配置します。

#### C. 有効化の確認方法

配置後、AIアシスタントに以下のように尋ねてください。
> 「今あなたが使えるスキルの一覧を教えて」
> 「`[スキル名]` の指示内容を確認して、実行の準備ができているか教えて」

このように返答があり、スキルの内容（`SKILL.md` の記述）をAIが把握していれば有効化成功です。

## 2. 他の端末でさらにスキルを収集・抽出する方法

別のPCや開発端末を使い始めた際、過去にそのPCで作成した「野良スキル」がローカルに眠っている場合があります。それらを本リポジトリへかき集める手順です。

### 2.1 捜索場所の目安

各種AIコーディングツールやプラグインが自動・手動で生成するスキルディレクトリは、通常以下の場所に隠れています。

- **ユーザーホーム内の隠しフォルダ** （最有力）
  - Windows: `C:\Users\<ユーザー名>\.codex\skills\` または `~/.antigravity` 関連
  - Mac/Linux: `~/.codex/skills/`
- **過去の開発プロジェクトのルートディレクトリ**
  - 例: `<開発プロジェクト>\skills\` や `<開発プロジェクト>\.agents\` 等

### 2.2 一括検索コマンドを利用する

PC内からスキルフォルダ（名前が skill 関連のもの）を効率よく探し出すには、OSごとのコマンドを活用します。

**Windows (PowerShell) の場合**:

以下のコマンドを実行すると、ユーザーフォルダ配下の不要なキャッシュ（AppData等）を除外しつつ一覧を出力します。
```powershell
Get-ChildItem -Path "$HOME" -Filter "*skill*" -Recurse -Directory -ErrorAction SilentlyContinue | Where-Object { $_.FullName -notmatch "AppData" } | Select-Object FullName
```

**Mac / Linux の場合**:

```bash
find ~ -type d -name "*skill*" 2>/dev/null | grep -v "Library"
```

### 2.3 収集と集約の手順

1. 上記の検索で見つかった「プロジェクト特有の指示書ではなく、汎用的に使えるスキル（例: `frontend-skill`、`pdf-reader` など）」を選別します。
2. それらのフォルダを、本リポジトリの `agent` ディレクトリ内にコピーします。
3. `git add .` → `git commit -m "新しいスキルを他端末から収集"` → `git push` を行います。
4. メインの開発端末に戻り、`automation/auto_update.py` （または `git pull`）を実行して同期完了です！

### 2.4 スキルの統合・整理（集約ルール）

分散したスキルを本リポジトリに集約する際は、以下のルールを適用することで、最新性の維持と過去の資産の保護を両立させます。

1. **同一内容のスキップ**
   - 収集先で見つかった `SKILL.md` の内容が、本リポジトリの既存のものと完全に一致する場合は、統合処理をスキップします。

2. **複数バージョンの退避（バックアップ）**
   - すでに本リポジトリに同名のスキルが存在し、かつ内容が異なる（古い、あるいは別バリエーションである）場合は、既存のフォルダを `old/` ディレクトリ内に退避させます。
   - 退避時のフォルダ名例: `old/[スキル名]_[タイムスタンプ]`
   - これにより、常に `agent/` や `skills/` には最新（またはメイン）のスキルが配置されるようにします。

3. **インデックスの更新**
   - 新しいスキルを追加した後は、必ず `SKILL_INDEX.md` を更新し、適切なカテゴリ（`text`, `medical`, `automation`, `development` 等）に登録してください。

> [!IMPORTANT]
> スキル内にAPIキーや個人のクレデンシャル、特定端末の直書き絶対パス（例: `C:\Users\Itaru Akaza\...`）が含まれていないかを、集約（共有）する前に必ず確認・修正してください。
